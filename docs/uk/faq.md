# Часті запитання (FAQ)

Якщо ви не знайшли відповідь на своє запитання, створіть issue на [GitHub](https://github.com/svitrix/svitrix-firmware/issues).

## Загальне

#### З: Чи працює SVITRIX без системи розумного дому?

В: Так, частково. SVITRIX має вбудовані застосунки для часу, дати, температури, вологості та рівня батареї — вони працюють одразу після прошивки без будь-якої зовнішньої системи.

Однак SVITRIX насамперед розроблений як компаньйон розумного дому. Для повного розкриття можливостей — кастомні застосунки, сповіщення, віддалене керування — потрібна зовнішня система (HomeAssistant, IOBroker, NodeRed тощо), яка взаємодіє через MQTT або HTTP API.

#### З: Чи можу я використовувати інші розміри матриці, окрім 32x8?

В: Ні. SVITRIX оптимізований для Ulanzi TC001, який використовує матрицю 32x8 пікселів (256 LED). Інші розміри не підтримуються.

## Прошивка та встановлення

#### З: Отримую помилку під час прошивки через SVITRIX Web Flasher

В: Спробуйте інший USB-кабель та інший USB-порт. Деякі кабелі (включаючи комплектний) підтримують лише зарядку, але не передачу даних. Якісний дата-кабель — найпоширеніше вирішення цієї проблеми.

#### З: У мене лише 192KB вільної флеш-пам'яті. Що не так?

В: Стокова прошивка Ulanzi використовує іншу таблицю розділів з меншим розділом файлової системи.

Щоб отримати повний обсяг ~1350KB доступного місця, використовуйте [онлайн-прошивальник SVITRIX](./flasher) — він записує всі розділи, включаючи файлову систему. OTA-оновлювач Ulanzi записує лише розділ програми та залишає стару таблицю розділів на місці.

## Дисплей та іконки

#### З: Виникають графічні збої з іконками

В: GIF-рендерер SVITRIX підтримує 8-бітні GIF. Якщо ви помітили графічні збої, ваш GIF, ймовірно, має прозорість. Замініть прозорі пікселі на суцільний чорний колір за допомогою онлайн-інструменту, наприклад [цього](https://onlinegiftools.com/add-gif-background).

#### З: Блимає індикатор у куті дисплея. Що це означає?

В: SVITRIX використовує невеликі LED-індикатори у кутах матриці для сигналізації про проблеми з підключенням:

| Індикатор | Позиція | Значення |
|-----------|---------|----------|
| 1 | Верхній правий кут | Проблема з WiFi |
| 2 | Нижній правий кут | Проблема з MQTT-брокером |

Після відновлення підключення індикатор зникає автоматично.

## Налаштування

#### З: Що таке `DoNotTouch.json` і що робити, якщо я випадково змінив цей файл?

В: `DoNotTouch.json` зберігає критичні налаштування пристрою:

- Конфігурація статичної IP-адреси
- Налаштування MQTT-підключення
- Префікс HomeAssistant Discovery
- NTP-сервер / часовий пояс
- Пароль HTTP Auth

Якщо ви випадково змінили цей файл, видаліть його через [файловий менеджер](./webinterface) та перезавантажте пристрій. Файл буде створено заново автоматично, але вам потрібно буде повторно налаштувати перераховані вище параметри.

#### З: Датчик температури показує завищене значення

В: Датчик в Ulanzi TC001 розташований всередині герметичного корпусу. Внутрішнє тепло від LED-матриці (залежно від яскравості) та акумулятора підвищує показання вище реальної кімнатної температури.

Компенсувати це можна за допомогою зміщень у файлі `dev.json` через [файловий менеджер](./webinterface):

```json
{
  "temp_offset": -5,
  "hum_offset": -1
}
```

Перезавантажте пристрій після збереження. Підберіть значення під ваше середовище — точне зміщення залежить від налаштувань яскравості та використання батареї.

Усі доступні налаштування розробника описані в розділі [Приховані функції](./dev).

## Саморобні збірки

#### З: Хочу зібрати власний SVITRIX. Яку прошивку використовувати?

В: Використовуйте [прошивальник SVITRIX](./flasher) — прошивка сумісна як з Ulanzi TC001, так і з саморобними збірками на базі будь-якої плати ESP32-WROOM (включаючи ESP32 D1 Mini).

Переконайтеся, що ви дотримуєтесь правильної схеми підключення з [Апаратного посібника](./hardware).

#### З: Саморобний пристрій показує незрозумілі символи на матриці

В: Потрібно змінити тип розкладки матриці. За замовчуванням використовується `0`, який може не відповідати вашій схемі підключення.

Створіть файл `dev.json` через [файловий менеджер](./webinterface) та спробуйте значення `1` або `2`:

```json
{
  "matrix": "1"
}
```

| Значення | Розкладка |
|----------|-----------|
| 0 | Одна матриця 32x8, рядкова, зигзаг |
| 1 | Чотири панелі 8x8 (4x1), рядкова, послідовна |
| 2 | Одна матриця 32x8, колонкова, зигзаг |

Перезавантажте пристрій після збереження.

## OTA-оновлення

#### З: Як оновити прошивку без USB?

В: Відкрийте [веб-інтерфейс](./webinterface) у браузері, перейдіть до розділу оновлення та завантажте новий файл `firmware.bin`. Пристрій перезавантажиться автоматично після оновлення.

Останню версію прошивки можна завантажити зі сторінки [GitHub Releases](https://github.com/svitrix/svitrix-firmware/releases).
