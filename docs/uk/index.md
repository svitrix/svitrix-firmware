::: info Відмова від відповідальності
Це програмне забезпечення з відкритим вихідним кодом не пов'язане з компанією Ulanzi та не схвалене нею. Використання програмного забезпечення здійснюється на ваш власний ризик, і я не несу відповідальності за будь-які потенційні збитки або проблеми, що можуть виникнути внаслідок його використання. Важливо розуміти, що це ПЗ з відкритим кодом не має прямої підтримки чи гарантій від компанії Ulanzi.
:::

SVITRIX — це кастомна прошивка для [Ulanzi Smart Pixel clock](https://www.ulanzi.com/products/ulanzi-pixel-smart-clock-2882?ref=28e02dxl) (реферальне посилання).
Вона створена як компаньйон для вашого розумного дому — HomeAssistant, IOBroker, NodeRed тощо.

Прошивка готова до роботи одразу після встановлення — з попередньо налаштованими застосунками часу, дати, температури та вологості. Вам не потрібно нічого робити, окрім увімкнення пристрою.
Однак для досвідчених користувачів можливості кастомізації через власні застосунки дозволяють розкрити весь потенціал SVITRIX.

::: warning
У SVITRIX термін «Застосунки» не означає традиційні додатки для смартфона, які ви завантажуєте та встановлюєте. Натомість CustomApps функціонують як динамічні сторінки, що обертаються в циклі відображення. Ці сторінки не зберігають і не виконують власну логіку — вони відображають контент, надісланий із зовнішньої системи, наприклад розумного дому. Контент передається через MQTT або HTTP протоколи за допомогою [CustomApp API](./api#custom-apps-and-notifications).
Важливо зазначити, що вся логіка управління контентом має оброблятися вашою зовнішньою системою. SVITRIX лише надає платформу для відображення інформації. Ви можете оновлювати контент у реальному часі в будь-який момент.
:::

SVITRIX — це форк спільноти, заснований на оригінальному проєкті [AWTRIX 3](https://github.com/Blueforcer/awtrix3). Рішення створити форк було прийнято для активної розробки та прийняття внесків від спільноти.

## AI-First розробка

SVITRIX розробляється за підходом **AI-first**. [Claude](https://claude.ai) від Anthropic є основним інструментом розробки — від архітектурних рішень і генерації коду до рефакторингу, тестування та управління CI. Проєкт використовує [Claude Code](https://github.com/anthropics/claude-code) як основний інструмент розробки, з кастомними хуками та скілами, адаптованими під embedded firmware workflow.
